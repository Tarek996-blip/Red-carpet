# دار السجاد — موقع متجر متكامل (Full-Stack)

موقع متجر سجاد كامل: واجهة أمامية + API + لوحة تحكم لإدارة المنتجات، مبني بـ Node.js و Express، بدون قاعدة بيانات معقدة (تخزين JSON بسيط) عشان يكون سهل التشغيل والنشر المجاني.

## المحتويات
- `server.js` — السيرفر و الـ API
- `db.js` — التعامل مع قاعدة البيانات (ملف data/db.json)
- `public/` — الواجهة (index.html للمتجر, admin.html للوحة التحكم)
- `public/uploads/` — صور المنتجات المرفوعة

## 1) التشغيل على جهازك محليًا

1. ثبّت [Node.js](https://nodejs.org) (نسخة 18 أو أحدث).
2. افتح Terminal داخل مجلد المشروع ونفّذ:
   ```
   npm install
   cp .env.example .env
   ```
3. افتح ملف `.env` وغيّر:
   - `ADMIN_USERNAME` و `ADMIN_PASSWORD` (بيانات دخولك للوحة التحكم)
   - `JWT_SECRET` (أي نص عشوائي طويل)
4. شغّل السيرفر:
   ```
   npm start
   ```
5. افتح المتصفح على: `http://localhost:3000` (المتجر) و `http://localhost:3000/admin` (لوحة التحكم).

## 2) تعديل بيانات المحل
- غيّر رقم الواتساب في `public/js/main.js` (متغير `WHATSAPP_NUMBER`) وفي `public/index.html` (رابط واتساب وزر الاتصال).
- غيّر اسم المتجر والنصوص في `public/index.html`.
- أضف منتجاتك الحقيقية من لوحة التحكم `/admin` (احذف المنتجات التجريبية الموجودة).

## 3) رفع الموقع مجانًا (Hosting) — أسهل طريقة: Render.com

1. اعمل حساب مجاني على [render.com](https://render.com) (تقدر تسجل بحساب GitHub).
2. ارفع مجلد المشروع على GitHub (لو محتاج مساعدة في الخطوة دي قولّي).
3. من لوحة Render: **New > Web Service** واختر الـ repo بتاعك.
4. الإعدادات:
   - **Build Command:** `npm install`
   - **Start Command:** `npm start`
   - **Plan:** Free
5. من تبويب **Environment**، ضيف نفس متغيرات `.env`:
   - `ADMIN_USERNAME`, `ADMIN_PASSWORD`, `JWT_SECRET`, `PORT=10000` (Render بيحدد البورت تلقائيًا غالبًا، سيبه كما هو لو ظهرت مشكلة جرب 10000)
6. اضغط Deploy، وهتاخد رابط زي: `https://your-store.onrender.com`

> ⚠️ ملاحظة مهمة: في الخطة المجانية من Render، أي ملفات بترفعها (صور المنتجات) أو تعديلات على `data/db.json` ممكن تتمسح لو السيرفر اتعمله إعادة تشغيل أو نشر جديد، لأن التخزين مؤقت (ephemeral). دي مش مشكلة وانت بتجرب، لكن للاستخدام الفعلي على المدى الطويل، الأفضل بعدين ننقل تخزين الصور والبيانات لخدمة خارجية مجانية زي Cloudinary (للصور) و Supabase/MongoDB Atlas (لقاعدة البيانات) — وأنا أقدر أساعدك تعمل الخطوة دي لما الموقع يبقى جاهز للانطلاق الفعلي.

### بدائل استضافة مجانية أخرى
- **Railway.app** — خطوات شبيهة بـ Render.
- **Fly.io** — يحتاج خبرة أكبر شوية لكن فيه تخزين دائم مجاني محدود.

## 4) "رفع الموقع على جوجل" — يعني إيه بالظبط؟

فيه معنيين محتملين، ودا الفرق:

**أ) ظهور موقعك في نتائج بحث جوجل (SEO):**
بمجرد ما يبقى عندك رابط حقيقي (زي اللي هتاخده من Render)، تقدر:
1. تعمل حساب مجاني على [Google Search Console](https://search.google.com/search-console)
2. تضيف رابط موقعك وتثبت ملكيتك له
3. تطلب من جوجل فهرسة (Index) الموقع
هتلاقي موقعك يظهر في نتائج البحث خلال أيام لأسابيع حسب نشاط الموقع.

**ب) استضافة الموقع على سيرفرات جوجل (Google Cloud):**
دي طريقة استضافة تانية (بديلة لـ Render) باستخدام Google Cloud Run، فيها خطة مجانية محدودة لكن إعدادها أعقد شوية. لو دا اللي قصدته بالظبط قولّي وهظبطلك الإعدادات اللازمة (Dockerfile وغيره).

## 5) أمان مهم قبل ما تنشر فعليًا
- غيّر `ADMIN_PASSWORD` لكلمة سر قوية.
- غيّر `JWT_SECRET` لنص عشوائي طويل وغير متوقع.
- متشاركش ملف `.env` مع حد ولا ترفعه على GitHub (موجود بالفعل في `.gitignore`).
